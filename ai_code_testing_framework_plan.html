Companion to Testing the Machine
ValidAI — Web-Based Testing Framework
for AI-Generated Code
An interactive platform that lets developers upload, analyse, and systematically test AI-generated code against the failure modes catalogued in the book. Each module maps directly to a chapter, turning principles into executable tests.
Open source Browser-native No install required 20 test modules Book-aligned
Core Architecture
📥
Code Ingestion Layer
Paste code, upload files, or connect a GitHub repo. Detects language, strips non-code, extracts function signatures and docstrings for downstream analysis.
⚙️
Analysis Engine
Runs selected test modules in parallel via Web Workers. Each module targets a specific AI failure mode and returns structured findings with severity scores.
📊
Results & Report Layer
Aggregates findings into a scored report. Exports as JSON, PDF, or GitHub comment. Links every finding back to the relevant book chapter and fix pattern.
Test Modules — one per book chapter
FailureMode Scanner
Static + pattern
Identifies AI-characteristic bugs: off-by-one boundaries, semantic drift, silent exception swallowing, and missing edge case branches. Uses AST analysis + regex pattern library curated from Chapter 3's bug zoo.
Ch 3
Oracle Checker
Specification diff
Compares code behaviour against a user-supplied spec written in plain English or JSON schema. Flags where the code's observable outputs diverge from stated requirements. Highlights the oracle problem in practice.
Ch 4
Prompt Testability Score
Prompt analysis
Given a generation prompt alongside the generated code, rates testability on five axes: dependency injection present, error semantics explicit, pure function ratio, edge cases specified, constraint completeness.
Ch 6
Property Generator
Auto test synthesis
Infers testable properties from function signatures and docstrings. Generates Hypothesis (Python) or fast-check (JS) test stubs covering invariants, round-trips, and algebraic relations. Runs them in a sandboxed executor.
Ch 7
Differential Runner
Cross-model compare
Takes two implementations of the same function — AI-generated vs reference, or two different models — and runs them against a shared input corpus. Highlights disagreements and auto-shrinks to minimal failing cases.
Ch 8
Complexity Profiler
Performance analysis
Detects algorithmic complexity red flags: nested loops, unbounded recursion, O(n²) sort usage, full table scan patterns in ORM code, memory accumulation. Runs synthetic benchmarks at 10×, 100×, 1000× input size.
Ch 9
Security Probe
OWASP + AI patterns
Scans for the vulnerabilities AI consistently generates: SQL injection via string interpolation, weak cryptography, JWT validation gaps, session management flaws, and prompt injection surfaces. Maps findings to OWASP Top 10.
Ch 13
Mutation Scorer
Test suite quality
Imports an existing test suite alongside the generated code. Applies semantic mutations (boundary shifts, operator flips, return type changes) and reports what percentage the tests catch. Surfaces coverage blind spots.
Ch 7 / 10
Hallucination Detector
API existence check
Extracts all method and function calls in generated code and cross-references them against published package documentation. Flags calls to non-existent APIs, wrong parameter counts, and deprecated signatures.
Ch 2
AI Review Assistant
LLM-powered
Uses Claude to review the submitted code specifically for AI-generated anti-patterns: omniscient functions, cargo-cult design patterns, leaky abstractions, inconsistent naming. Returns structured findings with fix suggestions.
Ch 11
User Workflow
1
Submit code + context
Paste generated code or link a GitHub file. Optionally attach the generation prompt and an existing test suite. Language is auto-detected; context improves scoring accuracy.
Supports: Python · TypeScript · JavaScript · Java · Go · C#
2
Select test profile
Choose a preset — Quick Scan (2 min), Security Focus, Full Audit — or hand-pick individual modules. Each module shows its estimated run time and what failure mode it targets.
Presets align with the book's risk-based testing strategy (Ch 5)
3
Run analysis
Modules execute in parallel via Web Workers. A live progress panel shows each module's status and streams findings as they arrive — no waiting for a full run to see first results.
Sandboxed execution via Pyodide (Python) · QuickJS (JS) · WASM
4
Review findings
Each finding links to the relevant book chapter, explains why it matters, and suggests a fix. Severity is rated Critical / High / Medium / Info. Findings are grouped by category, not by line number.
Inline code diff · chapter deep-link · copy-paste fix pattern
5
Export and iterate
Export findings as a structured JSON report for CI integration, or as a PDF for team review. Resubmit updated code to track improvement. Historical runs build a quality trend chart.
GitHub Actions integration · SARIF output · webhook support

Key Screens
Dashboard
Landing view showing recent submissions, quality trend over time, and aggregate stats across the codebase.
  • Quality score trend (7-day sparkline)
  • Finding breakdown by severity
  • Module pass/fail heatmap
  • Recent submissions list
Code Submission
Split-pane editor — code on the left, prompt + context on the right. Module selector with time estimates below.
  • Syntax-highlighted editor (CodeMirror)
  • Prompt field with token counter
  • Profile picker: Quick / Security / Full
  • Drag-and-drop file upload
Live Analysis View
Module progress shown as they run. Findings stream in per-module; critical findings rise to the top immediately.
  • Per-module status spinners
  • Streaming findings feed
  • Severity-filtered live list
  • Cancel / re-run individual modules
Finding Detail
Drilled-in view for a single finding. Code diff, explanation, book chapter link, and one-click fix pattern.
  • Inline before/after diff
  • Deep-link to book chapter
  • Copy fix pattern to clipboard
  • Mark as accepted / false positive
Report Export
Structured export for CI pipelines, team reviews, or compliance documentation. Multiple formats.
  • JSON (machine-readable, SARIF v2.1)
  • PDF (human review)
  • GitHub PR comment template
  • Markdown summary
Trend History
Quality score over time for a function, file, or whole repo. Shows how iterative prompt improvement affects output.
  • Score over time by module
  • Comparison across submissions
  • Highlight when prompt changed
  • Export to CSV
